## Challenges You May Encounter

### Challenge 1: File Size & String Replacement
**Issue:** Large files can cause `replace_string_in_file` failures

**Solution (from learned experience):**
- If replacement fails 2-3 times, create supplementary file
- Format: `module-X-sections-Y-Z-EXPANDED.md`
- Document in task list and inform human expert
- Example: Module 1 used this approach for sections 9-15

### Challenge 2: Maintaining Consistency
**Issue:** Keeping terminology and style uniform across modules

**Solution:**
- Reference Modules 1-2 frequently
- Use same equation notation conventions
- Match section structure (Introduction, Fundamentals, Design, Examples, Specs, Verification)
- Maintain same depth (~10,000 words per major section)

### Challenge 3: Realistic Example Values
**Issue:** Avoiding generic academic values

**Solution:**
- Reference real manufacturers: THK, HIWIN, SKF, Siemens, etc.
- Use standard component sizes: HGR15/20/25 rails, √ò16/20/25mm screws
- Check industry catalogs for typical specifications
- Verify calculations yield "round but realistic" results
